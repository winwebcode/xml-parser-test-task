Тестовое задание для Backend разработчика.


Целью данного ТЗ является: продемонстрировать соискателем умения работать с парсингом XML-файла, ORM и простыми HTTP запросами
Возможно использование одного из PHP фреймворков (Laravel,  Symfony). Приблизительное время на выполнение в часах – 3-5.
Дано.
XML файл с выгрузкой уцененных товаров из 1с – Trade In markdown.xml
Задание.
Используя нативные библиотеки PHP, распарсить XML файл в базу данных (СУБД на усмотрение соискателя). НЕ задавать полям в БД кириллические имена.


Проблематика.
В исходном файле не хватает необходимых для вывода на сайт данных - name, images и original_price. Используя GET запрос к эндпойнту https://stage.api.iport.ru/api/products и передавая в него КодТовара из файла, получить недостающие данные (пример запроса). В случае ненахождения товара, или отсутствия в ответе необходимых данных, записать товар из XML-файла в БД как неактивный с именем, равным значению поля КодТовара. Соответствие обязательных полей и данных из ответа:
'name' => 'TITLE',
'images' => 'IMAGES',
'original_price' => 'PRICE.VALUE'


Ожидаемый результат.
Кодовая база состоящая из класса/библиотеки классов, реализующая парсинг данных в БД с использование дополнительного запроса к API. 


Дополнительно.
Предусмотреть гибкое обновление данных в БД – при следующих выгрузках из файла (парсинге) обновлять имеющиеся данные, если они уже есть в базе и деактивировать те товары, которые есть в базе, но которых нет в файле. Осуществлять это по полю КодТовара (считаем, что оно никогда меняться не будет, все остальные поля могут меняться). 
Выполнение этого пункта на усмотрение соискателя.
